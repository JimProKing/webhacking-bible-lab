# 01-basics : 웹 해킹이란? + 왜 웹인가? (책 Chapter 01)

이 챕터는 기술보다는 **마인드셋**과 **근본 원인**을 다룹니다.

## 책에서 말하는 핵심 포인트 (요약)

1. 웹 해킹 = 웹 애플리케이션에 대한 공격
2. 웹 서비스가 공격 대상 1순위인 이유
   - 방화벽을 뚫고 들어올 필요가 거의 없음 (80/443 포트는 거의 항상 열려있음)
   - 사용자 입력값을 **반드시** 받을 수밖에 없는 구조
   - 입력값 검증을 제대로 안 하면 바로 취약점
3. 대부분의 취약점은 "사용자 입력값에 대한 검증 미흡"에서 시작
4. 공격 기법만 외우지 말고, **프로그램이 어떻게 동작하는지** 이해해야 구조적인 취약점을 찾을 수 있음

## 이 실습 앱으로 확인할 것

`app.py`를 실행하고 아래를 해보세요.

### 실습 1: 다른 사람의 "비밀 메모" 보기
```powershell
cd 01-basics
python app.py
```

브라우저에서:
- `http://127.0.0.1:5000/memo?owner=sumi` 접속
- 수미의 비밀 고민이 보이는가?

### 실습 2: 관리자 플래그 탈취
- `?owner=admin` 으로 시도
- 플래그가 보이면 성공 (`플래그{webhacking_is_fun_with_input_validation}`)

### 실습 3: 코드로 원인 찾기
`app.py`의 `view_memo()` 함수를 열어서 다음을 확인:

```python
owner = request.args.get("owner", "chulsu")
c.execute("SELECT * FROM memos WHERE owner = ?", (owner,))   # ← 여기까지는 그럭저럭
# 하지만 "이 요청을 보낸 사람이 owner의 메모를 볼 권한이 있는가?" 에 대한 검사가 전혀 없음
```

**이 한 줄의 부재**가 취약점의 시작입니다.
