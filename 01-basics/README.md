# 모듈 01 — 왜 웹이 공격에 취약한가? (Ch01 기초 마인드셋)

**실습 교재 내 위치**: 문제 01

이 모듈은 기술을 배우기 전에 **"웹 서비스가 왜 해커들의 1순위 타겟인가"**를 가장 단순한 형태로 체감하는 데 목적이 있습니다.

---

## 학습 목표

이 모듈을 마치면 다음을 할 수 있습니다.

- "사용자 입력값 검증 미흡"이 정확히 무엇을 의미하는지 한 문장으로 설명할 수 있다.
- 권한 검사가 전혀 없는 파라미터가 얼마나 위험한지 직접 경험한다.
- 나중에 배울 IDOR, SQL Injection 등의 모든 취약점이 결국 이 한 가지 원인에서 출발한다는 것을 이해한다.

---

## 책(Ch01)에서 강조하는 핵심 메시지

1. 웹 서비스는 80/443 포트가 항상 열려 있고, 사용자 입력을 반드시 받아야 하는 구조다.
2. 대부분의 취약점은 "이 입력을 받은 사람이 이 데이터를 볼/수행할 권한이 있는가?"를 검사하지 않는 데서 시작한다.
3. 공격 기법만 외우지 말고, 프로그램이 어떻게 동작하는지를 이해해야 구조적인 취약점을 찾을 수 있다.

---

## 실습 진행 방법

1. 모듈 실행
   ```powershell
   cd 01-basics
   python app.py
   ```
   접속: http://127.0.0.1:5000

2. 정상 동작 관찰
   - `?owner=chulsu` 로 철수의 메모를 확인한다.

3. 공격 실습 (문제 01의 본 실습)
   - `?owner=sumi` 로 수미의 비밀 메모를 본다.
   - `?owner=admin` 로 관리자 플래그를 탈취한다.

4. 원인 분석
   - `app.py`의 `view_memo()` 함수를 열어서 다음 부분을 확인:
     ```python
     owner = request.args.get("owner", "chulsu")
     c.execute("SELECT * FROM memos WHERE owner = ?", (owner,))
     ```
   - **문제**: "현재 요청자가 이 owner의 메모를 볼 권한이 있는가?"에 대한 검사가 전혀 없다.
