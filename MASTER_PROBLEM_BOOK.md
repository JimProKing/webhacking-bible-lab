# 크리핵티브의 한 권으로 끝내는 웹 해킹 바이블
## 실습 교재 상세 가이드 (Curriculum & Workbook)

**이 문서는 이 실습 교재의 공식 학습 가이드이자, 문제집의 상세 설명서입니다.**

`/lab` 페이지를 실제 "문제 푸는 공간"으로 사용하고, 이 문서를 "교사용/학습자용 가이드"로 활용하세요.

---

## 1. 이 교재의 교육 철학

기존 웹 해킹 자료들은 대부분 "기술 나열 + 따라하기" 형식입니다.  
이 교재는 다릅니다.

- **문제 → 시도 → 실패 → 깨달음 → 풀이 → 방어** 의 사이클을 강제로 경험하게 만듭니다.
- 한국의 수험서/문제집 문화처럼 **"문제 01." "문제 02."** 형식으로 번호를 매겼습니다.
- 모든 문제는 `lab_problems.py`에 객체로 정의되어 있어, 형식과 난이도가 일관됩니다.
- 공격에 성공하는 것뿐만 아니라, **"왜 이 코드가 취약했는가?"**와 **"어떻게 고쳐야 하는가?"**를 반드시 연결합니다.

이 철학이 지켜질 때, 책 한 권을 "읽은 사람"이 아니라 "쓸 수 있는 사람"이 됩니다.

## 2. 전체 학습 로드맵

### PART 01 — 준비 (문제 01~03)
**목표**: 기술 이전에 "왜 웹이 공격에 취약한가"와 "HTTP를 내 손으로 다루는 근육"을 만든다.

- **문제 01 (Ch01)**: 파라미터 변조로 권한 없이 정보 탈취
  - 학습 포인트: 입력값 검증 미흡의 가장 단순한 형태
  - 자가진단: "이 요청을 보낸 사람이 이 데이터를 볼 권한이 있는가?"라는 검사가 있는지 없는지 구분할 수 있는가?

- **문제 02 (Ch02)**: curl + 브라우저 DevTools로 HTTP 직접 만들기
  - 학습 포인트: Burp 이전에 HTTP를 "손으로" 느끼는 것
  - 자가진단: 쿠키를 직접 조작해 세션을 하이재킹할 수 있는가?

- **문제 03 (Ch02)**: Burp Suite로 모든 트래픽을 가로채고 반복 공격
  - 학습 포인트: Repeater가 진짜 해킹의 본체임을 체감
  - 자가진단: 브라우저 주소창으로만 공격하는 습관이 완전히 사라졌는가?

**이 단계에서 포기하거나 대충 넘기면, 이후 SQL Injection부터 고생합니다.**

### PART 02 — 공격 (문제 04~13)
**목표**: 책에서 가장 비중이 큰 공격 기법들을 실제로 재현하고, 그 원리를 몸으로 이해한다.

#### SQL Injection (문제 04~08) — 이 책의 핵심
- 문제 04: 로그인 우회 (`admin' -- `의 맛)
- 문제 05: UNION SELECT로 테이블 전체 털기 (secrets 테이블 FLAG 수집)
- 문제 06: ID 기반 SQLi (가장 현실적인 형태)
- 문제 07: Boolean Blind (한 글자씩 참/거짓으로 추출)
- 문제 08: Time-based Blind (응답 지연으로 데이터 유추)

**Ch04 정복 기준**: 위 5가지 기법을 Burp Repeater 없이도 설명하고 재현할 수 있어야 합니다.

#### 기타 주요 공격 (문제 09~13)
- 문제 09: OS Command Injection
- 문제 10: Stored XSS (댓글을 통한 영구 실행)
- 문제 11: CSRF (토큰 부재 상태 변경)
- 문제 12: 파일 다운로드/업로드 취약점 (Path Traversal 포함)
- 문제 13: IDOR + Broken Access Control (프로필/관리자 페이지 우회)

### PART 03 — 방어 (문제 14~15)
**목표**: 공격으로 배운 것을 방어로 연결한다. "공격 알아야 방어 보인다"의 진짜 의미를 체감.

- **문제 14**: SQL Injection 방어 실습
  - VulnBoard `app.py`의 모든 `f"..."` 쿼리와 `execute_vulnerable_query`를 찾아서 Prepared Statement로 고친다.
  - 고친 후 원래 페이로드가 먹히는지 직접 확인.

- **문제 15**: 종합 방어 실습
  - Command Injection, XSS, CSRF, 파일, 접근제어 등 모든 취약점 위치를 찾아서 방어 코드를 설계한다.
  - 가능하면 직접 "secure 버전"을 만들어 보는 것을 추천.

## 3. 한 문제당 추천 학습 흐름 (교육적으로 가장 효과적인 방법)

1. **책 읽기** (해당 챕터의 이론 부분)
2. **문제 카드 읽기** (`/lab` 또는 `/problem/N`)
3. **Burp Repeater로 최소 20~30번 공격 시도**
4. **에러 메시지, 응답 변화, 시간 차이 등을 모두 기록**
5. **성공하거나 정말 막혔을 때만 힌트 → 풀이 순으로 확인**
6. **풀이를 읽은 직후, "이 취약점의 근본 원인은 한 줄로 무엇인가?"를 적어보기**
7. **안전한 코드 스니펫을 보고, "내가 지금 고칠 수 있는가?" 자문하기**
8. **체크박스로 완료 처리 + 다음 문제로 이동**

> **중요**: 풀이를 먼저 보지 마십시오.  
> "아 이게 안 되네..." 하면서 40분을 소비한 사람이, 5분 만에 풀이 본 사람보다 훨씬 더 많이 배웁니다.

## 4. 자가 평가 체크리스트 (전체)

### 기초 단계 (문제 01~03 완료 후)
- [ ] "입력값 검증 미흡"을 한 문장으로 설명할 수 있다.
- [ ] HTTP 요청을 curl이나 fetch로 직접 작성할 수 있다.
- [ ] Burp Repeater에서 요청을 복제 → 수정 → 재전송하는 것이 자연스럽다.

### SQL Injection 단계 (문제 04~08 완료 후)
- [ ] 로그인 폼에서 `admin' -- ` 로 우회할 수 있다.
- [ ] UNION SELECT로 컬럼 수를 맞추고 데이터를 추출할 수 있다.
- [ ] Boolean Blind와 Time-based Blind의 차이를 설명하고, 각각 한 글자 추출 로직을 작성할 수 있다.
- [ ] "이 쿼리가 실제로 서버에서 어떻게 생겼을까?"를 상상하면서 공격하는 습관이 생겼다.

### 종합 공격 단계 (문제 09~13 완료 후)
- [ ] Command Injection, Stored XSS, CSRF, 파일 관련, IDOR을 각각 한 번 이상 성공시켰다.
- [ ] 각 공격이 어떤 상황에서 주로 발생하는지 실제 서비스 예시를 들 수 있다.

### 방어 단계 (문제 14~15 완료 후)
- [ ] VulnBoard 소스에서 취약한 부분을 모두 찾아 설명할 수 있다.
- [ ] 각 공격 기법에 대해 "이 코드를 이렇게 바꾸면 안전하다"는 예시를 최소 1개씩 제시할 수 있다.
- [ ] "Defense in Depth(다중 방어)"의 의미를 실제 코드 관점에서 설명할 수 있다.

이 체크리스트의 대부분에 체크가 되어 있다면, 이 책의 실전 부분은 상당히 정복한 상태입니다.

## 5. 자주 하는 실수와 조언

- Burp를 켜지 않고 진행하기 → 절대 금지. 이 교재의 70%는 Burp를 통해 배우는 것입니다.
- 풀이를 너무 빨리 보기 → 30분 규칙을 지키세요.
- 성공한 공격을 저장하지 않기 → Repeater 탭 이름을 "05-union-secrets"처럼 명확히 붙이세요.
- 코드 수정 없이 "이해했다"고 생각하기 → Part 03은 반드시 손으로 고쳐봐야 합니다.
- DB를 초기화하지 않고 계속 실험하기 → `/reset`은 친구입니다.

## 6. 파일 구조와 확장 방법

### 주요 파일 역할
- `vulnboard/lab_problems.py` — 모든 문제의 정의. 여기만 수정하면 `/lab`과 `/problem` 페이지가 자동으로 업데이트됩니다.
- `vulnboard/app.py` — 실제 취약 로직이 들어있는 메인 앱. VULNERABLE 주석을 적극 활용하세요.
- `vulnboard/templates/lab.html` — 문제집 목록 화면 (진행도, 카드 UI)
- `vulnboard/templates/problem_detail.html` — 개별 문제 상세 + 풀이 reveal

### 문제 추가하고 싶을 때
1. `lab_problems.py`의 `PROBLEMS` 리스트에 새 dict 추가
2. 필요한 경우 VulnBoard에 새로운 취약 엔드포인트 추가
3. `/lab` 페이지가 자동으로 반영
