# 크리핵티브의 한 권으로 끝내는 웹 해킹 바이블
## 실습 교재 (Hands-on Workbook)

**한 권으로 끝내는 웹 해킹 바이블**의 내용을 **직접 해킹하면서 완전히 체화**하기 위한 교육용 실습 교재입니다.

이 교재는 단순한 예제 모음이 아닙니다.  
**한국식 문제집 스타일**로 설계된 15개의 체계적인 문제(문제 01 ~ 문제 15)를 순서대로 풀어가면, 책의 핵심 이론을 실전 감각으로 전환할 수 있도록 구성되어 있습니다.

---

## 📘 이 실습 교재의 학습 목표

이 교재를 끝까지 마스터하면 다음을 할 수 있게 됩니다.

- 웹 애플리케이션의 구조적 취약점이 발생하는 근본 원리를 코드와 실제 공격으로 설명할 수 있다.
- SQL Injection의 모든 주요 기법(인증 우회, UNION, Boolean Blind, Time-based)을 실전에서 직접 구사할 수 있다.
- XSS, CSRF, Command Injection, 파일 업로드/다운로드, IDOR 등 주요 웹 공격 기법을 이해하고 재현할 수 있다.
- 공격 성공 후, 해당 취약점을 막기 위한 시큐어 코딩 방법을 제시할 수 있다.
- Burp Suite를 단순한 도구가 아니라, "생각하는 과정"의 일부로 자연스럽게 사용할 수 있다.

> **핵심 철학**: "공격을 해야 방어가 보인다."  
> 이 교재는 책을 읽는 것만으로 끝나지 않고, **직접 부숴 보고 고쳐 보는** 경험을 통해 실력을 쌓는 것을 목표로 합니다.

## 🚀 시작하기 (3단계)

### 1. 준비물
- Burp Suite Community Edition (필수)
- Python 3 (Flask 환경)
- 브라우저 (Chrome/Firefox 추천)
- 이 교재 (이 README + `/lab` 페이지)

### 2. 실행
```powershell
cd vulnboard
python app.py
```

브라우저에서 `http://127.0.0.1:5002` 접속 후,  
상단 네비게이션의 **📘 정복 문제집** 버튼을 클릭하세요.

### 3. 학습 방법
1. 책의 해당 챕터를 먼저 읽습니다.
2. `/lab` 페이지에서 **문제 N** 카드를 엽니다.
3. 진입점 링크를 클릭하고 **Burp Repeater로만** 공격합니다.
4. 성공 기준을 달성하면 체크박스로 완료 처리합니다.
5. 막히면 힌트를 보고, 그래도 안 되면 "풀이 보기"를 엽니다.
6. 풀이를 읽은 후에는 반드시 **안전한 코드 스니펫**을 확인합니다.

## 📖 이 교재의 특징 (교육 설계 의도)

- **문제 → 풀이 구조**: 한국의 전통적인 문제집처럼 "문제"와 "풀이"를 명확히 분리했습니다. 풀이는 최대한 늦게 보도록 설계되어 있습니다.
- **객체화된 문제**: 모든 문제가 `lab_problems.py`에 데이터로 정의되어 있어, 일관된 형식으로 학습할 수 있습니다.
- **실습 중심 + 방어 연결**: 단순 공격 성공이 아니라, "이걸 어떻게 막을까?"까지 생각하게 만듭니다.
- **진행도 추적**: 브라우저 localStorage로 완료 현황을 저장하여, 어디까지 했는지 한눈에 확인 가능합니다.
- **Burp 강제 훈련**: 모든 실습이 Burp Repeater를 통해 이루어지도록 유도합니다.

## ✅ 완료 기준 (자가 평가)

이 교재를 완전히 끝냈다고 할 수 있는 기준은 다음과 같습니다.

- [ ] 문제 01~03을 통해 기초 마인드셋과 HTTP/Burp 조작 근육을 길렀다.
- [ ] 문제 04~08 (SQL Injection)을 통해 에러 기반, UNION, Blind 기법을 모두 직접 성공시켰다.
- [ ] 문제 09~13을 통해 다양한 공격 기법을 최소 1회 이상 재현했다.
- [ ] 문제 14~15에서 VulnBoard의 소스 코드를 직접 열어 방어 코드를 작성/이해했다.
- [ ] 각 문제의 풀이를 읽고, "이 취약점의 근본 원인은 무엇이고, 어떻게 막아야 하는가?"를 타인에게 설명할 수 있다.

- 이 프로젝트는 **교육 목적의 안전한 실습 환경**입니다. 실제 서비스나 타인의 시스템에 절대 사용하지 마십시오.

- 풀이를 너무 빨리 보지 마십시오. 30분 이상 고민한 후에 여는 것이 진짜 학습입니다.
