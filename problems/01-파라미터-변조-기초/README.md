# 문제 01. 파라미터 변조로 다른 사람의 비밀 메모 탈취

**책 매핑**: Chapter 01. 웹 해킹에 대한 이해 (1.2 해커들의 공격 맛집, 웹)

**난이도**: ★☆☆☆☆

---

## 학습 목표

이 문제를 통해 다음을 체화합니다:

- 웹 서비스가 사용자 입력을 "반드시" 받을 수밖에 없는 구조라는 점
- "입력값 검증 미흡"이 정확히 무엇을 의미하는지
- 권한 체크 없이 파라미터만으로 다른 사람의 데이터를 보는 것이 얼마나 쉬운지
- 이후 모든 공격(SQLi, IDOR, XSS 등)의 근본 원인이 결국 이 한 가지에서 출발한다는 사실

---

## 문제 상황

당신은 철수입니다.  
수미와 admin의 "비밀 메모"를 보고 싶습니다.  
로그인도 필요 없습니다. 단지 URL의 파라미터만 바꾸면 됩니다.

## 실습 타겟

이 문제 전용 미니 앱이 `problems/01-파라미터-변조-기초/app.py` 에 있습니다.

**실행 방법**:
```powershell
cd problems/01-파라미터-변조-기초
python app.py
```

접속: http://127.0.0.1:5000

## 미션 (순서대로 수행)

1. 정상 동작 확인  
   `?owner=chulsu` 로 철수의 메모를 본다.

2. 공격 1  
   `?owner=sumi` 로 수미의 비밀 메모(고백 관련 내용)를 확인한다.

3. 공격 2 (성공 기준)  
   `?owner=admin` 로 관리자의 플래그 `플래그{webhacking_is_fun_with_input_validation}` 를 탈취한다.

## 성공 기준

- admin의 플래그 문자열이 화면에 명확히 보인다.
- "왜 이게 가능한가?"를 한 문장으로 설명할 수 있다.

## 힌트

- 브라우저 주소창에서 `?owner=xxx` 부분만 바꿔보세요.
- 이 앱에는 로그인, 세션, 권한 체크가 전혀 없습니다.
- 서버 코드는 `owner = request.args.get("owner")` 뒤에 아무런 검증 없이 DB 조회를 합니다.
