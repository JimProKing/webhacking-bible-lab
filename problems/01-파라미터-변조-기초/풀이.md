# 문제 01 풀이

## 왜 이 공격이 성공했는가?

서버 코드의 핵심 (app.py):

```python
owner = request.args.get("owner", "chulsu")
c.execute("SELECT * FROM memos WHERE owner = ?", (owner,))
```

문제점:
- `owner` 파라미터를 **아무런 권한 검사 없이** 그대로 WHERE 절에 사용
- "지금 이 HTTP 요청을 보낸 사람이, 이 owner의 메모를 볼 **권한이 있는가?**" 에 대한 검사가 **전혀 없다**

이것이 책 Ch01이 가장 강조하는 **"사용자 입력값에 대한 검증 미흡"** 의 가장 단순하고 강력한 예시입니다.

## 보안하게 고치려면?

```python
owner = request.args.get("owner", "chulsu")

# 추가해야 할 최소한의 검증
current_user = get_current_user()  # 세션에서 가져옴

if owner != current_user.username and not current_user.is_admin:
    return "권한이 없습니다.", 403

# 그 후에 조회
```

더 나은 방법:
- 소유자 기반 조회가 아니라, **현재 로그인한 사용자의 데이터만** 조회 (WHERE user_id = session_user_id)
- 또는 명시적인 권한 체크 로직 추가
- 가능하면 숫자 ID 대신 UUID 사용 (예측 불가능하게)

## 이 한 가지 개념이 중요한 이유

이후에 배우는 거의 모든 웹 공격의 뿌리입니다:

- SQL Injection: 입력이 쿼리에 그대로 들어감
- IDOR: ID 파라미터를 검증 없이 사용
- Broken Access Control: "나는 admin이다" 라는 신호를 클라이언트가 보내는 것을 서버가 믿음
- XSS: 입력이 출력에 그대로 나감

Ch01을 대충 넘기면 나머지 모든 챕터가 기술 암기가 됩니다.
