# 문제 14. SQL Injection 방어 실습 (코드로 고치기)

**책 매핑**: Chapter 12. SQL 인젝션 취약점 방어 (12.1 원인과 대응, 12.2 시큐어 코딩 실습)

**난이도**: ★★☆☆☆

**실습 타겟**: `app.py` 소스코드 (공격이 아니라 방어)

---

## 학습 목표

- 지금까지 성공시켰던 모든 SQLi 공격의 **근본 원인**을 코드에서 직접 찾아내기
- "이 한 줄 때문에 모든 게 뚫렸다"는 것을 몸으로 느끼기
- Prepared Statement로 실제 고쳐보고, 고친 후 공격이 안 먹히는지 검증

## 미션

1. 저장소 루트의 `app.py` 를 열고 `# VULNERABLE:` 주석이 붙은 모든 곳 찾기
   - board 검색
   - post/<id>
   - login
   - search
   - execute_vulnerable_query 함수 등

2. f-string이나 문자열 결합으로 된 쿼리를 모두 `?` placeholder 방식으로 변경

3. 변경 후:
   - `/reset` 으로 DB 초기화
   - 문제 04~08의 페이로드를 다시 시도
   - **더 이상 우회/추출이 안 되는지 확인**

## 성공 기준

- 원래 성공하던 `admin' -- ` 페이로드가 로그인 우회를 못 함
- UNION 페이로드가 더 이상 secrets를 못 가져옴
- "이게 Prepared Statement의 힘"을 체감

## 중요한 점

공격을 배운 사람이 방어를 가장 잘 할 수 있습니다.
