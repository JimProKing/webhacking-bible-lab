# 문제 14 풀이

## 취약한 코드의 전형 (지금까지 우리가 공격한 부분)

```python
# login
query = f"SELECT * FROM users WHERE username = '{username}' AND password = '{password}'"

# board 검색
query = f"SELECT ... WHERE title LIKE '%{keyword}%' ..."

# post id
query = f"SELECT ... WHERE p.id = {post_id}"
```

모두 **사용자 입력이 문자열로 직접 쿼리에 들어감**.

## 올바른 수정 패턴

```python
# login
c.execute(
    "SELECT * FROM users WHERE username = ? AND password = ?",
    (username, password)
)

# board
c.execute(
    "SELECT ... WHERE title LIKE ? OR content LIKE ?",
    (f"%{keyword}%", f"%{keyword}%")
)
```

## 왜 이게 강력한가?

- DB 드라이버가 입력을 "데이터"로 취급 (쿼리 구조와 분리)
- 따옴표, -- , UNION 등의 메타문자가 의미를 잃음
- 에러 기반, UNION, Blind 모두 무력화

## 추가로 할 수 있는 방어

- 에러 메시지 사용자에게 자세히 보여주지 않기
- 최소 권한 DB 계정 사용
- WAF (보조 수단)
- ORM 사용 (가능한 경우)
