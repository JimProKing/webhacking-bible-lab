PART 01 — 웹 해킹 공격과 방어를 위한 기본적인 지식들

기본이 서야 이후 모든 공격/방어가 흔들리지 않습니다. 이 부분을 대충 넘기면 나중에 SQLi, XSS가 "기술 암기"로 끝납니다.

Chapter 01. 웹 해킹에 대한 이해 (왜 웹인가?)

책에서 배우는 핵심
  • 웹이 해커들의 공격 맛집인 진짜 이유 (80/443 포트, 사용자 입력 필수, 방화벽 무용지물)
  • 대부분의 취약점은 "사용자 입력값에 대한 검증 미흡"에서 시작
  • 공격 기법만 외우지 말고 프로그램 동작 원리를 이해해야 구조적 취약점을 찾음
  • 철수 → 수미 정보 탈취 예시 (파라미터 변조의 가장 단순한 형태)
✅ 이 챕터를 실습으로 정복하는 방법
  1. 01-basics 앱 실행
    cd 01-basics && python app.py → http://127.0.0.1:5000
    ?owner=sumi 와 ?owner=admin 로 다른 사람 비밀 메모/플래그 직접 탈취
  2. 02-http-burp 앱 실행
    cd 02-http-burp && python app.py → http://127.0.0.1:5001
    Burp Intercept ON 상태에서 ?user= 파라미터를 sumi, admin으로 바꿔서 정보 탈취 체험
  3. 코드로 원인 확인
    두 앱의 app.py에서 "권한 체크가 전혀 없는 DB 조회" 부분을 직접 찾아 읽기
자가 점검 (책 Ch01 정신)
□ "입력값 검증 미흡"이 정확히 무슨 의미인지 1문장으로 설명할 수 있는가?
□ 왜 웹 서비스는 사용자 입력을 "반드시" 받아야 하는 구조인가?
□ 이 예제에서 owner 대신 숫자 id를 썼다면 어떤 점이 달라질까? (나중에 IDOR)

VulnBoard 메인으로 (기초 복습 후 이동)

Chapter 02. 반드시 알아야 할 웹 해킹 기본 지식 (HTTP + Burp + SQL 기초)

책에서 배우는 핵심
  • HTTP 요청/응답의 모든 요소 (메서드, 헤더, 파라미터, 쿠키, 바디, 상태코드)
  • Burp Suite의 Proxy, Repeater, History, Decoder, Intruder 핵심 사용법
  • SQL 기본 문법 (SELECT, WHERE, UNION, ORDER BY, 주석, 문자열 처리)
  • 웹 아키텍처, 클라이언트/서버 사이드 스크립트 구분
✅ 이 챕터를 실습으로 정복하는 방법 (가장 중요! 반복 추천)
  1. 02-http-burp를 Burp와 함께 철저히 파괴
    - GET 파라미터 변조 (/profile?user=...)
    - POST 바디 확인 (로그인 폼)
    - 쿠키 탈취 및 세션 하이재킹 (/myinfo)
    - Repeater로 수십 번 변조 연습
    - /debug/request 로 요청 구조 확인
  2. VulnBoard에서 Burp 근육 키우기
    로그인, 게시판 검색, 글 보기 등 모든 동작을 Burp Repeater로만 수행하는 습관 들이기
  3. SQL 문법 직접 실험
    VulnBoard의 /debug/db 나 실제 에러 메시지 보면서 UNION, ORDER BY, 주석(--, #) 동작 확인
책의 강조점: "Burp 없이 웹 해킹을 한다는 건 상상도 할 수 없는 상태가 되어야 한다."
이 챕터를 제대로 안 하면 이후 Ch04 SQL Injection부터 고생합니다. 최소 3~4번 반복하세요.

PART 02 — 공격에 대한 이해와 공격 실습

Chapter 03. 정보 수집 (Reconnaissance)

책에서는 whois, Netcraft, Google Hacking (dorks), theHarvester, Nmap/Zenmap 등을 다룹니다.

현재 랩에서의 위치: 단일 타겟 실습이라 정보 수집 단계는 제한적입니다.
대신 VulnBoard에 들어가기 전에 Burp History와 /debug/db, robots.txt, 소스코드 주석, 에러 메시지 등을 통해 "수동 정보 수집"을 철저히 연습하세요.

실제 실무에서는 이 단계가 공격 시간의 60~70%를 차지합니다. 별도 실습으로 Nmap + Burp + Google dork를 병행 추천.

Chapter 04. SQL Injection 취약점 (이 책의 가장 큰 비중 + 이 랩의 메인)

책에서 배우는 핵심 (4.1~4.4)
  • SQL Injection의 발생 원리와 구조적 이유
  • 에러 기반, UNION 기반, Boolean Blind, Time-based Blind, Second-order
  • 컬럼 수 맞추기, 타입 맞추기, UNION SELECT 실전 방법론
  • 로그인 우회, 데이터 추출, 인증 우회 등 다양한 공격 벡터
✅ 이 챕터를 실습으로 정복하는 방법 (Level 1 → 5 순서대로)
Level 1: 로그인 우회 (가장 기본)
/login 바로가기
페이로드: admin' -- , ' OR '1'='1' --
성공 = admin으로 로그인 + /admin 접근
Level 2: 검색 + UNION (데이터 대량 탈취)
/board 검색 / /search
ORDER BY로 컬럼 수 맞추기 → UNION SELECT로 users/secrets 덤프
Level 3: 게시글 ID 기반 (현실 패턴)
게시판에서 글 클릭
/post/1' 로 에러 유발 → UNION 또는 Blind로 확장
Level 4~5: Blind SQLi (Boolean / Time-based)
/search 추천
SUBSTR + 참/거짓 또는 응답 지연으로 한 글자씩 추출.
FLAG{TIME_BASED_SQLi_MASTER} 획득이 목표
정복 완료 기준 (책 Ch04 완전 이해)
□ 모든 secrets 플래그 최소 3개 이상 직접 추출
□ Blind까지 한 글자씩 추출하는 논리를 설명할 수 있음
□ "이 쿼리가 서버에서 실제로 어떻게 생겼을까?"를 상상하면서 공격하는 습관이 몸에 밴 상태

추가로 해볼 것: /debug/db 로 실제 테이블 구조 확인하면서 "에러 메시지가 얼마나 많은 정보를 주는지" 체감하기.

Chapter 05. OS Command Injection

책 매핑: OS 커맨드 인젝션 이해 + 공격 방법론

실습 위치: /tools/ping

시도: 127.0.0.1 & whoami, 127.0.0.1 | dir, 127.0.0.1 & type vulnboard.db

Windows 환경이므로 &, |, && 등을 적극 활용. 명령어가 어떻게 연결되는지 Burp로 관찰하면서 이해.

Chapter 06. XSS + 세션 하이재킹

실습 위치: 아무 게시글 상세 페이지 (/post/1 등) → 댓글 작성

Stored XSS 테스트: <script>alert(document.cookie)</script>

더 나아가: 쿠키를 외부로 보내는 페이로드 작성 (이 랩에는 외부 서버가 없으니 console.log나 alert로 먼저 확인)

책에서 다루는 Reflected / Stored / DOM XSS 구분과 세션 하이재킹 연결을 여기서 맛보세요.

Chapter 07~11. CSRF, 파일 관련, 파라미터 변조, 접근 제한 미흡
Ch07 CSRF
현재 랩에 기본적인 CSRF 취약 예시가 부족합니다. (추가 예정)
대신 /admin의 사용자 삭제 기능이나 프로필 수정 등을 생각해보세요. 토큰이 없는 상태에서 다른 사이트의 폼으로 요청을 보낼 수 있는지 상상.
Ch08 파일 다운로드
/uploads/ 로 직접 파일 서빙하는 구조 자체가 Path Traversal 위험을 내포. /uploads/../../app.py 같은 시도 해보기.
Ch09 파일 업로드
/upload 페이지. 확장자 우회, 경로 조작, 업로드 후 실행까지 도전.
Ch10 파라미터 변조 + Ch11 URL 접근 제한 미흡 (IDOR / Broken Access Control)
/profile 과 /profile/admin, /admin 직접 접근.
일반 계정으로 로그인한 상태에서 Burp로 username 쿠키/파라미터를 조작하거나 SQLi로 is_admin=1 만들기.

PART 03 — 방어에 대한 이해와 방어 실습 (시큐어 코딩)

공격을 이해한 사람이 가장 잘 막을 수 있습니다. 이 부분은 "공격으로 배운 것을 방어로 연결"하는 것이 핵심입니다.

공통 방어 원칙 (책에서 반복 강조)
  • 모든 입력값은 신뢰하지 말고 검증 + 이스케이프 (Prepared Statement, ORM, Output Encoding)
  • 최소 권한 원칙 (현재 로그인한 사용자가 이 데이터/기능을 볼 권한이 있는가?)
  • CSRF 토큰, HttpOnly + Secure 쿠키, Content-Security-Policy 등
  • 에러 메시지는 사용자에게 자세히 보여주지 말기
각 취약점별 방어 포인트 (이 랩의 코드와 비교하며 공부)
공격이 랩의 취약 코드 위치방어 방향 (직접 생각해보기)
SQLi (Ch04 / Ch12) app.py의 execute_vulnerable_query, f-string 쿼리들 모든 쿼리를 ? placeholder + execute(param)로. 절대 문자열 연결 금지.
Command Injection (Ch05 / Ch13) /tools/ping 의 subprocess with shell=True shell=False + 명시적 인자 리스트, 또는 화이트리스트 검증.
XSS (Ch06 / Ch14) 댓글 content를 | safe 로 출력 저장 시 escaping 또는 출력 시 자동 escaping (Jinja 기본 동작 믿기). Html sanitizing.
Broken Access / IDOR (Ch10-11 / Ch18-19) /profile, /admin, /admin/delete 에서 세션 is_admin만 체크 서버에서 매 요청마다 "이 사용자가 이 리소스에 대한 권한이 있는가?" 재확인. 소유자 검증.
File Upload (Ch09 / Ch17) secure_filename만 사용 + 바로 /uploads/ 서빙 콘텐츠 타입 검사, 확장자 화이트리스트, 별도 저장소 + 비실행 경로, 랜덤 파일명.
추천 공부 방법:
1. VulnBoard의 app.py를 열고 # VULNERABLE: 주석이 붙은 부분을 모두 찾는다.
2. 그 부분을 "어떻게 고쳐야 안전할까?"를 직접 코드로 고쳐보거나, 머릿속으로 시뮬레이션.
3. 책 Part 03의 "시큐어 코딩 실습" 부분을 읽으면서 실제로 비교.

🏆 이 책을 정복했다는 의미 (최종 체크리스트)

  • □ Ch01~02 기초: Burp 없이 웹을 볼 수 없는 상태 + "입력값 검증 미흡"의 본질을 설명 가능
  • □ Ch04 SQLi: 로그인 우회 + UNION + Blind까지 직접 성공 + 원리를 설명 가능
